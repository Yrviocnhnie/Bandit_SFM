# 2026-04-01 Global Action Space Key Results

## Key Results

以下结果都在 global action space 下评估，重点保留当前最值得横向比较的几条主线。

## 统一评估设置

除“原始全局 action space 初始模型”外，下表都使用：

- `Plan B`: `scenario-stratified split`
- evaluator: `docs/scenario_recommendation_actions_v6.md`
- top-k: `3`

指标定义：

- `most_relevant`: top-3 中命中的 `most_relevant_3` 个数平均值
- `acceptable`: top-3 中命中的 `most_relevant_3 + plausible_3` 个数平均值
- `irrelevant`: top-3 中命中的 `irrelevant` 个数平均值

| 模型 | R/O most_relevant | R/O acceptable | R/O irrelevant | App most_relevant | App acceptable | App irrelevant |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| disjoint LinUCB（每个 sample 一个 ground-truth action，sample 3 most relevant 为 `50/25/25`） | 1.29 / 3 | 1.79 / 3 | 0.42 | 1.61 / 3 | 2.16 / 3 | 0.48 |
| disjoint LinUCB（`3+43`, `1.0 / 0.0`） | 1.64 / 3 | 2.19 / 3 | 0.07 | 2.03 / 3 | 2.52 / 3 | 0.20 |
| shared-param LinUCB v3（one-hot action id + stronger action features + 少量 context/action interaction） | 0.47 / 3 | 0.48 / 3 | 0.00 | 1.63 / 3 | 2.32 / 3 | 0.24 |
| NeuralEncoder LinUCB（UCB random init, `3+43`） | 2.88 / 3 | 2.92 / 3 | 0.02 | 2.95 / 3 | 2.98 / 3 | 0.01 |
| NeuralEncoder LinUCB（supervised-head warm start, `3+43`） | 2.89 / 3 | 2.93 / 3 | 0.02 | 2.95 / 3 | 2.98 / 3 | 0.01 |
| Direct NeuralUCB（end-to-end replay, hidden-feature UCB, `3+43`） | 0.41 / 3 | 0.95 / 3 | 0.25 | 0.97 / 3 | 1.82 / 3 | 0.43 |

## Takeaways

- 在 global action space 下，`disjoint LinUCB` 从“单 ground-truth action”切到 `3+43` 后，效果明显提升，说明 full-action supervision 是有效的。
- 当前 `shared-param LinUCB` 仍明显不如 `disjoint LinUCB`，说明在线性 shared 结构下，动作共享仍然过强。
- 两个轻量 neural bandit 版本都显著优于 `disjoint LinUCB 3+43`：
  - `NeuralEncoder LinUCB (UCB random init)`
  - `NeuralEncoder LinUCB (supervised-head warm start)`
- 这两种 neural 版本几乎打平，说明当前收益主要来自：
  - neural encoder 学到的更强 context representation
  - 而不一定是 UCB head 初始化方式本身
- `Direct NeuralUCB` 这一版明显失败：
  - `R/O` 和 `App` 都显著退化
  - top-3 基本塌缩到少数全局动作
  - 说明“直接用 bandit replay 同时更新 encoder + head”在当前离线 synthetic 设定下并不稳定

## NeuralEncoder LinUCB Data Ablation

下面这张表把 `NeuralEncoder LinUCB（supervised-head warm start）` 在不同训练数据设定下的结果，方便看数据设计本身的影响。

| 训练数据设定 | R/O most_relevant | R/O acceptable | R/O irrelevant | App most_relevant | App acceptable | App irrelevant |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `3 relevant only` | 2.65 / 3 | 2.76 / 3 | 0.09 | 1.83 / 3 | 2.35 / 3 | 0.38 |
| `3 relevant + 3 acceptable + N irrelevant` | 2.72 / 3 | 2.73 / 3 | 0.01 | 2.95 / 3 | 2.98 / 3 | 0.01 |
| `3 relevant + 43 other` | 2.89 / 3 | 2.93 / 3 | 0.02 | 2.95 / 3 | 2.98 / 3 | 0.01 |
| `3 relevant + 3 acceptable + N irrelevant + other` | 2.89 / 3 | 2.93 / 3 | 0.02 | 2.95 / 3 | 2.98 / 3 | 0.01 |
| `3 relevant + 3 acceptable + other` | 2.89 / 3 | 2.93 / 3 | 0.02 | 2.95 / 3 | 2.98 / 3 | 0.01 |

简要结论：

- `other=0` 很关键。`3 relevant only` 会明显退化，说明 neural encoder 也需要看到“其余 action 不该推”的信号。
- `3 relevant + 3 acceptable + other` 和 `3+43` 基本打平，甚至在 `R/O acceptable` 上略好一点。
- `hard negative` 在没有 `other=0` 时能把 `irrelevant` 压低，但会明显伤到 `R/O most/acceptable`。
- 当前最推荐的数据设定仍然是：
  - `3 relevant + 43 other`
  - 或 `3 relevant + 3 acceptable + other`

## NeuralLinear Sanity Check

针对 `NeuralLinear UCB v1`，额外做了两项 sanity check：

### 1. 确认 encoder pretraining 只使用 `Plan B` 的 train split

首次运行和 rerun 都满足：

- `train.raw.jsonl = 48,800`
- `test.raw.jsonl = 12,200`
- `train_test_overlap = 0`

并且：

- `ro_train_samples_expanded.jsonl = 2,244,800`
- 即 `48,800 x 46`

这说明：

- encoder 预训练使用的是 `Plan B` train split 展开得到的训练样本
- 没有把 test split 混进监督预训练

### 2. 同样配置 rerun 后结果稳定

首次运行：

- `R/O`: `2.8835 / 2.9262 / 0.0244`
- `App`: `2.9522 / 2.9775 / 0.0107`

rerun：

- `R/O`: `2.8848 / 2.9230 / 0.0238`
- `App`: `2.9516 / 2.9769 / 0.0109`

两次结果几乎重合，说明当前结果不是明显的偶然波动。

## New 2k / 65-Scenario Dataset

团队新提供的数据集：

- 输入文件：`bandit_v0_65_scenaroios_unique_support_to_2k_samples_office_working_alias.jsonl`
- 覆盖 `65` 个 scenario
- 每个 scenario 约 `2k+` raw samples
- split：`80/20 scenario-stratified`
- 训练设定：`3 relevant + 3 acceptable + N irrelevant + other`
  - `most_relevant = 1.0`
  - `plausible = 0.1`
  - `irrelevant = -0.1`
  - `other = 0.0`

说明：

- 原始新数据里有 `OFFICE_AFTERNOON`，而当前 relevance catalog 用的是 `OFFICE_WORKING`
- 为了和现有标注表对齐，这轮实验使用了一个 alias 版输入文件，把 `OFFICE_AFTERNOON -> OFFICE_WORKING`

| 模型 | R/O most_relevant | R/O acceptable | R/O irrelevant | App most_relevant | App acceptable | App irrelevant |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| disjoint LinUCB（`3+3+N irrelevant+other`） | 1.44 / 3 | 1.83 / 3 | 0.04 | 1.60 / 3 | 2.49 / 3 | 0.32 |
| NeuralEncoder LinUCB（UCB random init, `3+3+N irrelevant+other`） | 2.90 / 3 | 2.95 / 3 | 0.01 | 2.92 / 3 | 2.96 / 3 | 0.01 |
| NeuralEncoder LinUCB（supervised-head warm start, `3+3+N irrelevant+other`） | 2.89 / 3 | 2.94 / 3 | 0.01 | 2.92 / 3 | 2.95 / 3 | 0.01 |

简要结论：

- 在这份更大、覆盖 `65` 个 scenario 的新数据上，两个 NeuralEncoder LinUCB 版本仍然显著优于 `disjoint LinUCB`
- `UCB random init` 和 `supervised-head warm start` 依旧几乎打平
- 新数据下 `disjoint LinUCB` 的 `R/O` 退化更明显，而 neural 版本仍接近满分区间
- 从训练成本上看，`disjoint` 在大数据上明显更慢；这进一步支持后续优先沿 neural encoder 路线推进

## Current Recommendation

- 如果继续沿线性 bandit 主线推进，`3+43` 是当前 `disjoint LinUCB` 的最强训练数据设计。
- 如果目标是继续提升离线 top-3 相关性，并为后续 online learning 做准备，当前最值得继续推进的是轻量 neural 路线。
- 目前最推荐的默认起点是：
  - `NeuralEncoder LinUCB（supervised-head warm start）`
  - 因为它与最优结果持平，同时保留了“冻结 trunk，只在线更新最后一层 UCB head”的 bandit 结构。
