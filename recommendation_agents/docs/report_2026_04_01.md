# 2026-04-01 Bandit / Neural Bandit 进展汇报

## 概要

本轮实验主要做了两类事情：

- 继续在 `disjoint LinUCB` 上优化训练数据设计，重点测试了：
  - `3+3+2`
  - `3+3+40`
  - `3+43`
  - `3+3+38+2 hard negative`
  - `extra hard negative`
- 在相同 `Plan B scenario-stratified split` 上，尝试了：
  - `shared-parameter LinUCB`
  - `NeuralLinear UCB v1`

当前最重要的发现有两个：

- 在 `disjoint LinUCB` 上，让模型对每个 context 看到完整的全局 action space，并给非目标 action 明确的 `0 reward`，效果明显优于原始 `3+3+2` 设定。
- `NeuralLinear UCB v1` 在当前最强的 `3+43` 训练数据设定上，离线评估结果显著优于 `disjoint LinUCB`，而且 rerun 后结果稳定。

## 统一评估设置

除“原始全局 action space 初始模型”外，下表都使用：

- `Plan B`: `scenario-stratified split`
- evaluator: `docs/scenario_recommendation_actions_v6.md`
- top-k: `3`

指标定义：

- `most_relevant`: top-3 中命中的 `most_relevant_3` 个数平均值
- `acceptable`: top-3 中命中的 `most_relevant_3 + plausible_3` 个数平均值
- `irrelevant`: top-3 中命中的 `irrelevant` 个数平均值

## 结论

### 1. 在 `disjoint LinUCB` 上，训练数据设计比 reward 微调更重要

相对于原始 `3+3+2`：

- `3+3+40` 和 `3+43` 都明显提升了 `R/O most_relevant` 和 `R/O acceptable`
- 说明让模型对每个 context 看到完整 action space，并明确知道“哪些 action 不是当前推荐目标”，是有效的

其中：

- `3+43` 是当前 `disjoint LinUCB` 下 `R/O most_relevant` 最强的版本
- `3+3+40` 在 `App` 上更稳一些

### 2. `hard negative` 有帮助，但需要克制

加入 `hard negative` 后：

- 能明显压低定义好的 `irrelevant` 动作
- 对 `App` 指标也有正向帮助

但结果也表明：

- `hard negative` 更擅长清理明显错误动作
- 不一定直接拉高 `R/O most_relevant`
- 如果加得过多，会开始伤到 `R/O most_relevant`

所以当前结论是：

- `hard negative` 应该少量、场景化使用
- 不适合无限扩大

### 3. `shared-parameter LinUCB` 当前版本不成功

无论是：

- hash-based action features
- stronger action features
- 再加少量 context/action interaction

结果都明显不如当前的 `disjoint LinUCB`。

当前判断：

- shared 方向本身未必错
- 但当前这几版线性 shared 结构仍然过弱
- 动作共享过头，难以稳定地区分 `R/O` 的细粒度 action

### 4. `NeuralLinear UCB v1` 当前是最强结果

当前初版设计：

- encoder: `306 -> 128 -> 32`
- 先用离线 expanded samples 做监督预训练
- 再把 `32` 维 latent 当作 `h(x)`，接 per-action linear UCB heads

在 `3+43` 训练数据设定下，`NeuralLinear UCB v1` 相比 `disjoint LinUCB 3+43` 有明显提升：

- `R/O most_relevant`: `1.64 -> 2.88`
- `R/O acceptable`: `2.19 -> 2.93`
- `R/O irrelevant`: `0.07 -> 0.02`
- `App most_relevant`: `2.03 -> 2.95`
- `App acceptable`: `2.52 -> 2.98`
- `App irrelevant`: `0.20 -> 0.01`

## NeuralLinear Sanity Check

针对 `NeuralLinear UCB v1`，额外做了两项 sanity check：

### 1. 确认 encoder pretraining 只使用 `Plan B` 的 train split

首次运行和 rerun 都满足：

- `train.raw.jsonl = 48,800`
- `test.raw.jsonl = 12,200`
- `train_test_overlap = 0`

并且：

- `ro_train_samples_expanded.jsonl = 2,244,800`
- 即 `48,800 x 46`

这说明：

- encoder 预训练使用的是 `Plan B` train split 展开得到的训练样本
- 没有把 test split 混进监督预训练

### 2. 同样配置 rerun 后结果稳定

首次运行：

- `R/O`: `2.8835 / 2.9262 / 0.0244`
- `App`: `2.9522 / 2.9775 / 0.0107`

rerun：

- `R/O`: `2.8848 / 2.9230 / 0.0238`
- `App`: `2.9516 / 2.9769 / 0.0109`

两次结果几乎重合，说明当前结果不是明显的偶然波动。

## 当前建议

- 如果继续沿线性 bandit 主线推进，`3+43` 是当前 `disjoint LinUCB` 的最强训练数据设计。
- 如果目标是继续提升离线 top-3 相关性，并为后续 online learning 做准备，当前最值得继续推进的是：
  - `NeuralLinear UCB v1`
