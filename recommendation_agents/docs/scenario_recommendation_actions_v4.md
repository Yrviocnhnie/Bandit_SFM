# Scenario-wise Recommendation Actions

## 中文说明

本文件基于 `default_rules.json` 中的原始 scenario-action 映射，并在同一 scenario 下补充了额外的 recommendation actions。

三类 recommendation 的定义：
- `R` = Reminder。面向用户的提醒信息，例如喝水、休息、准备、收尾等。
- `O` = Operation。由“用户可见消息 + 系统操作变量”组成的 recommendation，例如 `show_today_schedule` 或 `turn_on_dnd`。
- `A` = App recommendation。共有 10 个 `app_category` 可选，由“用户可见消息 + app_category”组成，用来表示当前场景下直觉上最适合的默认 app 类别；每个 scenario 指定 1 个默认 `A`。

文档摘要：
- scenario 总数：65（原有 46 个，新增 19 个）
- R&O action 总数：167（其中已有 63 个，新增 104 个；“新增”指不来自 `default_rules_hq.json`）
- 主表中 action 类型计数：`R` = 66，`O` = 101
- 当前每个 scenario 在主表中包含 >=2 个 `R/O`，并在单独的 A 表中包含 1 个默认 `A`（共 65 个）
- `isDefault` 列在主表中用于标记该 scenario 下默认推荐的 `R/O`，在 A 表中用于标记默认 app recommendation
- `A` 中使用的 `app_category` 遵循 `data_spec_for_rl.md` 中的 taxonomy. 可选的 app 类别共有 10 个：`social`、`productivity`、`entertainment`、`navigation`、`shopping`、`news`、`health`、`music`、`reading`、`game`

说明：
- `R` 和 `O` 一起构成主表中的 recommendation set，用来覆盖同一场景下多个不同但都有用的提醒或操作
- 主表中的 `isDefault` 列只用于标记默认推荐的 `R` 或 `O`；A 表中的 `isDefault` 用于标记默认 app recommendation
- 主表内每个 scenario 的 action 列举顺序为 `R -> O`；A 表单独列出每个 scenario 的默认 app recommendation；同一类型内部，原始 action 在前，新加 action 在后；`trigger` 顺序为 `none -> duration -> exit`。
- 对于 `A` 类 action，这里推荐的是 app 类别（`app_category`），而不是某一个具体 app。原因是具体 app 和更细粒度的 app-level actions 几乎是无限的，不适合直接作为当前文档里的动作集合。因此，应该在 UCB agent 之外设计一个额外的映射机制：一方面可以根据用户的 app 使用历史，把某个 `app_category` 映射到该用户在这个类别下最常用、最合适的具体 app；另一方面也可以允许用户直接指定每个 category 对应的偏好 app，例如 `music -> QQ 音乐`、`shopping -> 美团`、`navigation -> 高德地图`。比如，当当前 scenario 推荐 `music` 类 app 时，系统可以结合 app usage history，或直接使用用户预先设定的偏好映射，打开该用户最常使用或最偏好的音乐 app。

## English Notes

This file is based on the original scenario-action mapping in `default_rules.json`, and then adds extra recommendation actions where they are useful for the same scenario.

Summary:
- Total scenarios: 65 (46 existing + 19 newly added)
- Total `R&O` actions: 167 (`63` existing and `104` new; “new” means not from `default_rules_hq.json`)
- Action type counts in the main table: `R` = 66, `O` = 101
- Each scenario contains at least 2 `R/O` actions in the main table, plus 1 default `A` in the separate app table (65 total)
- The `isDefault` column marks the default `R/O` in the main table, and the default app recommendation in the app table
- `A` uses the `app_category` taxonomy defined in `data_spec_for_rl.md`. There are 10 available app categories: `social`, `productivity`, `entertainment`, `navigation`, `shopping`, `news`, `health`, `music`, `reading`, `game`

Recommendation type definitions:
- `R` = Reminder. A user-facing reminder message, such as drink water, wrap up, prepare, or rest.
- `O` = Operation. A user-facing message paired with a system-side operation variable, such as `show_today_schedule` or `turn_on_dnd`.
- `A` = App recommendation. There are 10 available `app_category` values. Each `A` is a user-facing message paired with an `app_category`, indicating the intuitively most suitable default app category for the current scenario; each scenario has 1 default `A`.

Notes:
- `R` and `O` together form the recommendation set for the scenario in the main table, covering multiple useful reminders or operations in the same context.
- In the main table, `isDefault` is used to mark the default `R` or `O`. In the app table, it marks the default app recommendation.
- Within each scenario in the main table, actions are listed in `R -> O` order. Default app recommendations are listed separately in the app table. Within the same type, original actions come first and newly added actions come later; `trigger` is ordered as `none -> duration -> exit`.
- For `A` actions, this document recommends app categories rather than specific apps. The reason is that the space of possible apps and finer-grained app-level actions is effectively unbounded, so it is not suitable to treat concrete apps as the fixed action set in this document. Therefore, there should be a separate mechanism outside the UCB agent that maps an `app_category` to the most appropriate concrete app for the current user. This mapping can be learned from app usage history, or it can be explicitly configured by the user for each category, for example `music -> QQ Music`, `shopping -> Meituan`, or `navigation -> Amap`. For example, if the current scenario recommends a `music` app, the system can either use historical usage patterns or the user's predefined preference mapping to open the most suitable music app.

Quick stats:
- Total scenarios: 65 (46 existing + 19 newly added)
- Total `R&O` actions: 167
- Number of `R`: 66
- Number of `O`: 101
- Each scenario has 1 default `A`
