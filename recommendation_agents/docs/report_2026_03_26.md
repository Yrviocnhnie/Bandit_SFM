# 2026-03-26 Bandit V0 进展汇报

## 概要

今天的主要结果是得到了一版新的 `v6_candidate` LinUCB 模型，训练配置为：

- `most_relevant / plausible / irrelevant rewards = 1.0 / 0.1 / 0.0`
- `most_relevant / plausible / irrelevant repeat = 1 / 1 / 1`
- `alpha = 0.0`
- `default_bonus = 0.0`
- `epochs = 1`

- 训练数据 定义并加入了  每个scenario 最合适（3），次合适（3），不合适（2）的 R/O推荐 和 APP推荐
- 对每个模拟数据 context/features，训练不同action，reward = 1.0 /0.1 / 0.0
- 模型对 最合适，次合适action推荐的命中率上升，对不合适action推荐降低。
- 后续需探索 除DisjointLinUCB外的模型，如 shared-parameter UCB 和 Neural UCB 看是否能够进一步提升离线训练的效果。 因为现在是需要从features理解场景，没有把scenario id直接给进来

## 结论

- `R/O` 的平均 `most_relevant` 命中数从 `1.29 / 3` 提升到 `1.38 / 3`。
- `App` 的平均 `most_relevant` 命中数从 `1.61 / 3` 提升到 `2.09 / 3`。
- `App` 的 `acceptable` 也明显提升，从 `2.16 / 3` 提升到 `2.61 / 3`。
- `R/O` 的 `acceptable` 从 `1.79 / 3` 降到 `1.63 / 3`，说明当前版本更强调最相关动作，但整体可接受覆盖略有下降。
- `irrelevant` 进入 top-3 的情况明显减少：
  - `R/O`: `0.42 -> 0.26`
  - `App`: `0.48 -> 0.10`
