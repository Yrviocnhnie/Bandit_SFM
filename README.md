# RL Python Prototype

This folder contains the Python prototype of the app-aligned Dual UCB recommender. It mirrors the current HarmonyOS app's state encoding, scenario matching, feature-space layout, and Dual-UCB recommendation structure so you can debug or train the model outside the app.

## What Each File Does

- `__init__.py`
  - Marks `rl/` as a Python package.

- `ucb_cpp_port.py`
  - Compatibility entrypoint for the split Python implementation.
  - Re-exports the main classes so older imports can still work.
  - Runs a small demo that prints:
    - matched `state_code`
    - matched state name
    - matched `scenario`
    - matched scenario name
    - top RO and APP predictions

- `dual_ucb_model.py`
  - Core Dual UCB model implementation.
  - Contains:
    - the MLP prior network
    - the UCB statistics for each arm
    - one head for RO actions
    - one head for APP categories
  - Supports:
    - random weight initialization
    - forward prediction
    - lightweight training step
    - reward updates for the bandit part
    - save/load as JSON
    - loading pretrained weight arrays parsed from the C++ header

- `feature_space.py`
  - Defines the app-aligned feature-space vocabulary and layout.
  - Loads constants from the app when possible, such as:
    - state codes
    - location classes
    - time/day/motion/phone/sound classes
    - scenario IDs
    - RO action IDs
    - APP action categories
  - Defines the final `726`-dimensional feature layout used by the Python encoder and model.

- `state_and_scenario.py`
  - Defines the main data structures used across the Python prototype.
  - Includes:
    - `PhysicalState`
    - `StateChainEntry`
    - `ContextSnapshot`
    - `ScenarioDefinition`
    - `ScenarioMatch`
  - Loads scenario definitions from the app's `default_rules.json`.
  - Reads bandit scenario/action defaults from the app catalog.
  - Also provides debug helpers for:
    - state names
    - scenario names
    - bilingual debug labels

- `rule_engine.py`
  - Python version of the app-side state encoder and scenario matcher.
  - Converts a `PhysicalState` into a canonical state code.
  - Checks rule conditions and state-chain preconditions against a `ContextSnapshot`.
  - Produces:
    - matched scenario
    - RO target distribution
    - APP target distribution
    - debug info for state/scenario names

- `ucb_input_encoding.py`
  - Python feature encoder for the Dual UCB model.
  - Converts a `ContextSnapshot` plus optional scenario ID into the shared `726`-dimensional feature vector.
  - Mirrors the app-side `DualBanditFeatureEncoder.ets` layout as closely as possible.
  - Includes helpers for:
    - one-hot encoding
    - tri-state encoding
    - scalar normalization
    - state-chain parsing
    - location normalization

- `random_training_features.py`
  - Synthetic sample generator for local training/debugging.
  - Creates randomized but app-shaped:
    - physical states
    - state chains
    - snapshots
    - matched scenarios
    - encoded features
    - RO/APP target distributions
  - Also attaches debug labels like state and scenario names to each sample.

- `train_runner.py`
  - Small command-line training script for the Python Dual UCB prototype.
  - Repeatedly:
    - samples synthetic data
    - trains the MLP prior
    - applies a small reward to the winning RO/APP arms
  - Saves the model as JSON and prints final predictions with debug state/scenario names.

- `dual_ucb_random_model.json`
  - Example saved model generated by `train_runner.py`.
  - Stores the trained Python model state in JSON format.

## Data Flow

The main pipeline is:

`PhysicalState / state chain` -> `RuleEngine` -> `ContextSnapshot` -> `DualBanditFeatureEncoderPy` -> `DualUCBModel`

More concretely:

1. `rule_engine.py` encodes the current physical context into a canonical state code.
2. `rule_engine.py` matches rules and state-chain transitions to a scenario.
3. `ucb_input_encoding.py` converts the snapshot and scenario into a `726`-dimensional feature vector.
4. `dual_ucb_model.py` predicts:
   - RO actions
   - APP categories

## Debugging Entry Points

- Run the demo:

```bash
python rl/ucb_cpp_port.py
```

- Run a short synthetic training loop:

```bash
python rl/train_runner.py --steps 10 --topk 3
```

The printed debug output includes:

- state code
- state name
- scenario ID
- scenario name
- top RO predictions
- top APP predictions

## Notes

- This Python prototype is aligned to the current app's rules and naming as much as possible by reading:
  - `default_rules.json`
  - `BanditActionCatalog.ets`
  - `DualBanditFeatureEncoder.ets`
  - `ContextModels.ets`

- Some online history/feedback features are still placeholder-filled on the Python side if the app-side live values are not exported into the prototype inputs yet.
